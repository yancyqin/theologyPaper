# Theology Paper debug checklist

This repository contains a checklist of key theological concepts, refinements, and insights for writing theology papers. The focus is on ensuring clarity, biblical accuracy, and theological precision in expression.

# Theological Writing Self-Check Framework

> 核心原则：所有判断必须结合上下文，而不是机械套用

| 分类 | 常见问题（错误方向） | 更合适表达（修正方向） | 检查问题 |
|------|----------------------|------------------------|-----------|
| 1. 视角 | 从人出发：construct, organize, analyze | 从神启示出发：revealed, given, made known | 这句话是人起点，还是神起点？ |
| 2. 认识论 | 强调理解：understand, broaden, deepen | 强调认识：know, receive, see, abide | 是在掌握知识，还是认识神？ |
| 3. 层级 | 神学被当作源头或生命来源 | God → Word/Spirit → theology（次序清晰） | theology 有没有越位？ |
| 4. 语感 | 学术化：consolidate, construct, participation, limited | 圣经语感：abide, stand firm, grounded, walk | 更像论文，还是更像圣经表达？ |
| 5. 经文使用 | 混合总结和经文语气，像经文但不是经文 | 要么引用原文，要么明确是总结 | 是圣经原意，还是我自己的话？ |
| 6. 神学 vs 生命 | 用系统解决奥秘：resolve, reconcile | 指向认识神：His ways are higher | 是在解题，还是在认识神？ |
| 7. Stronghold | 神学框架先入为主影响读经 | 让 Scripture 本身说话 | 是经文在说话，还是框架在说话？ |
| 8. 上下文一致性 | 各种视角混用但不自觉 | 根据段落功能切换视角 | 当前句子属于哪一层语境？ |

---

## 快速检查（写完一段用）

1. 起点：God → Word → Spirit，还是人 → 思考 → 系统？
2. 方向：认识神，还是理解概念？
3. 层级：theology 是否被抬高？
4. 语感：更像圣经，还是更像论文？
5. 经文：有没有“像经文但不是经文”？
6. 本质：是在解问题，还是在认识神？
